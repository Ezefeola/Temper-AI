# Token Budget — TemperAI Standards

## Philosophy

Token usage is a finite resource. Every phase of the SDD workflow consumes tokens proportional to its input context and output size. The budget system ensures:

1. **Visibility** — you always know how many tokens each phase consumed
2. **Control** — you can set hard limits per phase
3. **Optimization** — you can identify which phases are the most expensive
4. **Predictability** — you can estimate the total cost of a project before starting

## Budget Rules

- **Always estimate before executing** — the orchestrator estimates input and output tokens before spawning a phase.
- **Always check against budget** — if the estimated total would exceed the budget, warn the user before proceeding.
- **Always update after completion** — after a phase completes, update the actual usage in `.temper/budget.md`.
- **Always alert at 80%** — when budget utilization reaches 80%, warn the user.
- **Always stop at 100%** — when budget is exhausted, do not spawn more phases without explicit user override.

## How to Estimate Token Usage

### Input Tokens

Count the tokens in:
1. **System prompt** — the agent's instructions (from the `.agent.md` file)
2. **Skills loaded** — the combined size of all loaded skill files
3. **Context files** — the `.temper/` files the phase reads
4. **Source files** — the code files the phase needs to read

Rough estimation:
- 1 word ≈ 1.3 tokens
- 1 line of code ≈ 15 tokens
- 100 lines ≈ 1,500 tokens

### Output Tokens

Estimate based on the phase's expected output:
- **prd.md** — 1,500-3,000 tokens
- **specs/INDEX.md** — 500-1,500 tokens + **US-XXX-*.md** — 300-800 tokens each (depends on number of user stories)
- **Docs/domain-model.md** — 4,000-8,000 tokens (depends on number of entities and endpoints)
- **tasks/INDEX.md** — 500-1,500 tokens + **T###-*.md** — 200-500 tokens each (depends on number of tasks)
- **Code per task** — 500-2,000 tokens (depends on complexity)
- **Review report** — 1,000-3,000 tokens
- **Documentation** — 3,000-6,000 tokens

## Budget Optimization Tips

### Reduce input tokens

- Use the quick path for simple tasks (saves 80-95%)
- Load only the skills the phase needs
- Only pass relevant files, not the entire codebase
- Keep skill files concise — avoid verbose examples

### Reduce output tokens

- Keep phase outputs focused — no filler content
- Use tables instead of long paragraphs
- Reference existing files instead of repeating their content
- Skip sections that are not applicable

### Monitor and adjust

- Check `.temper/budget.md` after each phase
- If a phase consistently exceeds estimates, adjust the budget
- If a phase uses less than expected, note the savings
- Use the `temper-ai budget` CLI command for a quick overview
